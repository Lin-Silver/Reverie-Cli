# Reverie Modeling Pipeline

This project uses a Reverie-Gamer-only modeling workflow.

## Source Of Truth
- `assets/models/source/` stores authoring files such as `.bbmodel`.
- `assets/models/runtime/` stores engine-facing exports such as `.glb` and `.gltf`.
- `playtest/renders/models/` stores previews and review snapshots.
- `data/models/model_registry.yaml` is generated by the modeling workbench.

## Required External Apps
- Install Blockbench desktop manually.
- Install the Ashfox plugin inside Blockbench.
- Keep Blockbench running while using live Ashfox MCP tools through Reverie-Gamer.

## Built-In Reverie Integration
- Reverie seeds a built-in `ashfox` MCP server entry automatically.
- Ashfox MCP tools are exposed only in `reverie-gamer` mode.
- `/modeling` manages stubs, imports, registry sync, and stack inspection without local helper scripts or checked-out source references.

## Recommended Flow
1. Author or revise the `.bbmodel` source in Blockbench.
2. Use Ashfox MCP tools from Reverie-Gamer to validate, preview, and export runtime geometry.
3. Import the exported runtime model into `assets/models/runtime/`.
4. Run the registry sync step so Reverie Engine can discover the updated model set.
